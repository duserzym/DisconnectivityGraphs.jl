API

Data Model

DisconnectivityGraphs.Minimum — Type
Minimum(id, energy; metadata=Dict{Symbol,Any}())

A local minimum in an energy landscape. id can be any hashable label, and energy is usually an absolute energy in joules or a consistently shifted energy in dimensionless units.

source
DisconnectivityGraphs.Saddle — Type
Saddle(from, to, energy; metadata=Dict{Symbol,Any}())

An undirected transition-state connection between two minima. For systems with directed barriers, store the absolute saddle energy here and keep forward and reverse barriers in metadata, or construct this from the higher of E_minimum + barrier for the two directions.

source
DisconnectivityGraphs.LandscapeGraph — Type
LandscapeGraph(minima, saddles)

A sparse minima-transition-state network. The core disconnectivity-tree algorithm only requires minimum energies and saddle energies; metadata can hold micromagnetic moments, mesh names, NEB convergence metrics, or path profiles.

source
DisconnectivityGraphs.minimum_ids — Function
minimum_ids(landscape)

Return minimum ids in the stored order of a LandscapeGraph.

source
DisconnectivityGraphs.saddle_energy — Function
saddle_energy(saddle)

Return the energy value associated with a Saddle. This small helper is used as the default sorting key during merge-tree construction.

source

Tree Construction

DisconnectivityGraphs.DisconnectivityNode — Type
DisconnectivityNode

A node in a disconnectivity merge tree. Leaf nodes represent minima and internal nodes represent the saddle energy at which child basins merge.

source
DisconnectivityGraphs.DisconnectivityTree — Type
DisconnectivityTree

The merge tree returned by disconnectivity_tree. root indexes the top node in nodes, and leaf_for_minimum maps each minimum id to its leaf.

source
DisconnectivityGraphs.disconnectivity_tree — Function
disconnectivity_tree(landscape; link_disconnected=:root)

Build an exact merge tree by adding saddles from low to high energy. The method is the same connectivity idea used by disconnectivity graph codes such as pele: minima start as isolated leaves, and every saddle that first connects two previously disconnected basins creates a new branch node at the saddle energy.

Set link_disconnected=:error to reject disconnected transition networks. The default, :root, creates a final artificial root at the highest available energy so partially sampled landscapes can still be visualized.

source
DisconnectivityGraphs.component_partition — Function
component_partition(landscape, threshold)

Return the connected components of minima using only saddles with saddle.energy <= threshold.

source
DisconnectivityGraphs.leaf_order — Function
leaf_order(tree)

Return leaf minimum ids in the left-to-right order induced by the tree.

source

Layout

DisconnectivityGraphs.TreeSegment — Type
TreeSegment

A backend-independent line segment used to draw a disconnectivity tree. kind is :branch for vertical branch segments and :merge for horizontal merge segments.

source
DisconnectivityGraphs.TreeLayout — Type
TreeLayout

Plotting geometry returned by tree_layout, including node positions, leaf positions keyed by minimum id, and drawable tree line segments.

source
DisconnectivityGraphs.tree_layout — Function
tree_layout(tree; order=leaf_order(tree), leaf_spacing=1.0)

Compute backend-independent line segments for plotting a disconnectivity tree. The returned TreeLayout can be consumed by PlotlyJS, Makie, Plots, or a custom publication renderer.
